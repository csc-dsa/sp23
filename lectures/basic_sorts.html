Basic Sorting Algorithms#

Sorting#

Given \(n\) elements that can be compared according to a total order relation

  • we want to rearrange them in non-increasing/non-decreasing order

  • for example (non-decreasing):

    • input: sequence of items \(\ \ \ \ \ \ \ \ \ \ \ A = [k_0, k_1, \dots, k_{n - 1}]\)

    • output: permutation of A \(\ \ \ \ \ \ \ \ \ \ \ B\ |\ B[0]\ \le\ B[1]\ \le \dots B[n - 1]\)

\[\begin{split}\\ \\ \\ Central\ problem\ in\ computer\ science\end{split}\]

Insertion Sort#

Defined asâ€¦

Array is divided into sorted and unsorted parts

Invariants

  • elements in unsorted have not been seen

Pseudocode
 1void insertion_sort(int* items, int size) {
 2  //grows the left part (sorted)
 3  for (int i = 1; i < size; i++) {
 4    int j = i;
 5    //inserts j in sorted part
 6    while (j > 0 && items[j-1] > items[j]) {
 7      swap(items[j-1], items[j]);
 8      j--;
 9    }
10  }
11}

Running time depends on the input

Worst-case?#

input is already sorted in descending order

\[\begin{split}\begin {align} \sum_{i = 1}^{N - 1} i &= 1 + 2 + 3 + \dots + (N - 1) \\ &= \frac{N(N - 1)}{2} \\ &= O(N^2) \\ \end {align}\end{split}\]
Average-case?#

each element \(\approx\) halfway order
expect every element to move O(n/2) times

\[\begin{split}\begin {align} \sum_{i = 1}^{N - 1} \frac{i}{2} &= \frac{1}{2} (1 + 2 + 3 + \dots + (N - 1)) \\ &= \frac{N(N - 1)}{4} \\ &= O(N^2) \\ \end {align}\end{split}\]
Best-case?#

input is already sorted in ascending order

\[\begin{split}\begin {align} \sum_{i = 1}^{N - 1} 1 &= (N - 1) \\ &= O(N) \\ \end {align}\end{split}\]

Selection Sort#

Defined

Array is divided into sorted and unsorted parts

  • algorithm scans array from left to right

invariants

  • elements in sorted are fixed and in ascending order

  • no element in unsorted is smaller than any element in sorted

code
 1void selectionsort (int *A, unsigned int n) {
 2  int temp;
 3  unsigned int i, j, min;
 4  // grows the left part (sorted) 
 5  for (i = 0; i < n; i++) {
 6    min = i;
 7    // find min in unsorted part
 8    for (j = i + 1; j < n; j++) {
 9      if (A[j] < A[min]) {
10        min = j;
11      }
12    }
13    // swap A[i] and A[min]
14    temp = A[i];  
15    A[i] = A[min];
16    A[min] = temp;
17  }
18}
\[Number\ of\ comparisons?\ |\ Number\ of\ exchanges?\]
Notes

Selection Sort: moves through each element finding the minimum value and replacing the current element value as it goes.

\[A[0] \rightarrow A [1] \rightarrow A[2] \rightarrow A[n - 1]\]

  • sorted portion left order, fixed, untouched

  • unsorted iterated through to find remaining min value to swap current element value
